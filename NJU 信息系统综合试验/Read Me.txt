
###程序简介###
本程序用Django编写网页代码、Bootstrap3负责用户界面设计、MySql负责数据存储与处理，实现了TinyCute书店的
客户系统以及管理系统，粗略的包含了客户注册登录、购书；管理员处理顾客订单、下达采购单以及基本数据统计
分析功能。

###程序使用说明###
step1 安装python2.7，django以及xampp
step2 安装python的package manager "pip"，在cmd下运行"pip install MySQLdb"（安装mysql针对Python的接口）
step3 在phpMyAdmin下建立bookstore数据库（排序规则为UTF8-bin）
step4 在cmd下，切换至程序根目录，运行"python manage.py inspectdb"（将本地的Python models同步至mysql中）
step5 打开xampp的mysql以及apache服务
step6 在cmd下，运行"python manage.py runserver"
step7 打开127.0.0.1:8000即进入本系统的客户界面，用管理员账户密码登录之后则跳入管理员界面。

###客户系统简介###
	## 搜书模块 ##
		查询书店书籍的基本信息
	## 荐书模块 ##
		推荐书店购买相应图书
	## 购书模块 ##
		购买相应图书，下达购买订单。如若图书缺货，则提示客户该图书存货不足，并记录该缺货情况，当货源充足时再通知客户。
	## 个人信息管理模块 ##
		# 个人信息修改 #
			修改个人的基本信息
		# 图书订单查询 #
			查询图书订单的状态（处理与未处理）
		
###管理系统简介###
	## 采购部 ##
		# 供应商管理模块 #
			添加备选供应商记录
		# 采购单处理模块 #
			采购单处理（每周五为采购单处理日，采用轮盘赌选择相应供应商，采购价格为销售价格的60%，并且有20%机率采购单无法被满足）
		# 策略补货处理模块 #
			补货条件为“时间为每周五并且该图书库存<安全库存值”，补货量通过经济批量公式计算可得。（具体参数可见程序里的views.py源码）
	## 销售部 ##
		# 顾客订单处理模块 #
			对顾客订单进行处理
		# 客户信息管理模块 #
			对客户基本信息，购买信息进行管理，并按照客户消费额度递减排序
		# 图书库存查询模块 #
			书店图书现有库存查询
	## 财务部 ##
		# 库存帐模块 #
			显示目前各种图书库存情况
		# 收支表模块 #
			按日给出书店的具体收支、利润情况
		# 统计分析模块 #
			按照销售量、销售额排出前十以及后十的图书
